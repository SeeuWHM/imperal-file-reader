# Design history (archived)

These five documents are the original pre-implementation design/planning
notes for the File Reader upgrade (dated 2026-07-08 through 2026-07-10).
They predate the current code and are kept for historical context only —
**not** current documentation. Several of their checklist items are already
implemented (quality/extraction-method passthrough, `read_file_preview`,
bearer auth); others were superseded by different decisions made while
building. For what the extension actually does today, see the top-level
`README.md`.

- `AI-ONLY-ASSESSMENT.md` — early assessment of AI-only vs OCR extraction strategy
- `BACKEND-RECOMMENDATIONS-FOR-LLM-2026-07-10.md` — handoff notes for backend formatting
- `FILE-READER-API-CONTRACT.md` — original target API contract draft
- `FILE-READER-FEDERAL-GRADE-GUIDE.md` — original federal-grade implementation guide
- `FILE-READER-IMPLEMENTATION-CHECKLIST.md` — original execution checklist (companion to the guide above)
